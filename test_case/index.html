    10. Тест-кейсы для задач MVP

    10.0.1 User Story 1: Оформление заказа¶

    ID Название тест-кейса Описание Предусловия Шаги Ожидаемый результат
    TC1.1 Просмотр меню Проверка корректного отображения меню. Пользователь вошел в систему. 1. Открыть меню ресторана.
    2. Проверить наличие всех блюд и их описание.
    3. Проверить отображение цен и наличия скидок.    		 Меню отображается корректно, все блюда видны с полной информацией.
    TC1.2 Добавление блюда в корзину Проверка возможности добавления блюда в корзину. Пользователь просматривает меню. 1. Выбрать блюдо из меню.
    2. Указать количество порций.
    3. Нажать кнопку "Добавить в корзину".    		 Блюдо добавлено в корзину, количество порций отображается корректно.
    TC1.3 Оформление заказа Проверка процесса оформления заказа. Пользователь добавил блюда в корзину. 1. Перейти в корзину.
    2. Проверить состав корзины.
    3. Подтвердить заказ.
    4. Выбрать способ оплаты.
    5. Провести оплату.    		 Заказ оформлен, подтвержден и оплачен.
    TC1.4 Оплата заказа Проверка успешности оплаты заказа. Заказ оформлен. 1. Перейти к оплате заказа.
    2. Выбрать способ оплаты.
    3. Завершить процесс оплаты.    		 Оплата проходит успешно, заказ подтвержден.
    TC1.5 Подтверждение заказа Проверка получения номера заказа после оплаты. Оплата завершена успешно. 1. После завершения оплаты, проверить номер заказа (талон). Пользователь получает номер заказа, статус заказа изменен на "оплачен".

    10.1 Список тест-кейсов для задач первого релиза¶

    ID Название тест-кейса Описание Предусловия
    TC2.1 Просмотр карточки блюда Проверка возможности просмотра детальной информации о блюде. Пользователь просматривает меню.
    TC2.2 Проверка состава корзины Проверка отображения всех добавленных позиций и их цен в корзине. Пользователь добавил блюда в корзину.
    TC2.3 Удаление блюда из корзины Проверка возможности удаления блюда из корзины. Блюдо добавлено в корзину.
    TC2.4 Регистрация в программе лояльности Проверка успешности регистрации в программе лояльности. Пользователь не зарегистрирован в программе лояльности.
    TC2.5 Проверка баланса программы лояльности Проверка корректности отображения баланса бонусов. Пользователь зарегистрирован в программе лояльности.
    TC2.6 Отмена заказа Проверка возможности отмены заказа до его выполнения. Заказ еще не выполнен.
    TC2.7 Просмотр состояния роботов Проверка получения актуальной информации о состоянии роботов. Пользователь или менеджер по производству.

    10.1.1 Негативные тест-кейсы для задач MVP¶

    ID Название тест-кейса Описание Предусловия Шаги Ожидаемый результат
    TC1.6 Ошибка отображения меню Проверка реакции системы на ошибку отображения меню. Пользователь вошел в систему. 1. Открыть меню ресторана.
    2. Убедиться, что меню не отображается.
    3. Система должна предложить перезагрузку или обратиться к персоналу.    		 Сообщение об ошибке и предложение обратиться к персоналу.
    TC1.7 Невозможность добавить блюдо в корзину Проверка системы при попытке добавить недоступное блюдо. Пользователь просматривает меню. 1. Выбрать блюдо, которого нет в наличии.
    2. Попытаться добавить блюдо в корзину.
    3. Проверить сообщение об ошибке.    		 Сообщение о том, что блюдо недоступно.
    TC1.8 Пустая корзина при оформлении заказа Проверка системы при попытке оформить заказ с пустой корзиной. Пользователь добавил блюда в корзину. 1. Перейти в корзину.
    2. Удалить все блюда.
    3. Попытаться подтвердить заказ.
    4. Проверить сообщение об ошибке.    		 Сообщение о необходимости добавления хотя бы одного блюда.
    TC1.9 Ошибка оплаты заказа Проверка системы при неудачной попытке оплаты заказа. Заказ оформлен. 1. Перейти к оплате заказа.
    2. Попробовать оплатить заказ с невалидной платежной информацией.
    3. Проверить сообщение об ошибке и предложение выбрать другой метод оплаты.    		 Сообщение о неудачной оплате и предложение выбора другого способа.
    TC1.10 Ошибка отправки заказа в производство Проверка реакции системы на ошибку при отправке заказа в производство. Оплата завершена успешно. 1. Завершить оплату.
    2. Проверить статус заказа.
    3. Проверить сообщение об ошибке при отправке заказа в производство.    		 Сообщение о необходимости обращения к персоналу для решения проблемы.

    10.1.2 Негативные тест-кейсы для задач первого релиза¶

    ID Название тест-кейса Описание Предусловия Шаги Ожидаемый результат
    TC2.1 Ошибка при просмотре карточки блюда Проверка системы при попытке просмотра карточки недоступного блюда. Блюдо выбрано для просмотра. 1. Выбрать недоступное или удаленное блюдо.
    2. Попытаться открыть карточку блюда.
    3. Проверить сообщение об ошибке.    		 Сообщение о том, что карточка блюда недоступна.
    TC2.2 Некорректные данные в корзине Проверка реакции системы на некорректные данные в корзине. Блюдо добавлено в корзину. 1. Открыть корзину.
    2. Попытаться сохранить корзину с некорректными данными (например, отрицательное количество).
    3. Проверить сообщение об ошибке.    		 Сообщение о некорректных данных в корзине.
    TC2.3 Ошибка удаления блюда из корзины Проверка системы при попытке удалить недоступное или несуществующее блюдо. Блюдо есть в корзине. 1. Попытаться удалить блюдо, которое уже удалено или отсутствует в корзине.
    2. Проверить сообщение об ошибке.    		 Сообщение о том, что блюдо не найдено в корзине.
    TC2.4 Ошибка регистрации в программе лояльности Проверка системы при некорректных данных при регистрации в программе лояльности. Пользователь не зарегистрирован. 1. Попробовать зарегистрироваться с некорректными данными (например, неправильный формат почты).
    2. Проверить сообщение об ошибке.    		 Сообщение о некорректных данных регистрации.
    TC2.5 Отсутствие информации о балансе программы лояльности Проверка системы при попытке получить информацию о балансе несуществующего клиента. Клиент зарегистрирован в программе. 1. Попытаться получить информацию о балансе для клиента, которого нет в базе.
    2. Проверить сообщение об ошибке.    		 Сообщение о том, что клиент не найден.
    TC2.6 Ошибка отмены заказа Проверка системы при попытке отмены уже выполненного заказа. Заказ еще не выполнен. 1. Попробовать отменить заказ, который уже выполнен.
    2. Проверить сообщение об ошибке.    		 Сообщение о том, что заказ уже выполнен или не найден.
    TC2.7 Ошибка получения состояния роботов Проверка системы при попытке получения информации о состоянии роботов в случае их недоступности. Пользователь или менеджер по производству. 1. Попытаться получить информацию о состоянии роботов.
    2. Проверить сообщение об ошибке, если информация недоступна.    		 Сообщение о недоступности информации о состоянии роботов.
